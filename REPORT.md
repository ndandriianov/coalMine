# Отчёт по проекту CoalMine

## Суть приложения

**CoalMine** - симулятор угольной шахты, реализованный в виде полноценного веб-приложения с разделением
на бэкенд и фронтенд. Пользователь запускает шахту, нанимает шахтёров трёх типов (малый, средний, сильный),
покупает оборудование (кирка, вентиляция, вагонетки) и наблюдает за добычей угля в реальном времени.
Каждый шахтёр работает в отдельной горутине и добывает уголь с разной производительностью и запасом энергии;
помимо этого шахта приносит пассивный доход. Добытый уголь является валютой: за него нанимаются новые рабочие
и покупается снаряжение. Доступны управляющие действия: старт, стоп, рестарт, пауза и возобновление - все они
отражаются на состоянии горутин в реальном времени.

## Технологии

### Vue 3

Vue 3 — один из наиболее популярных фреймворков для написания веб-приложений.

Моменты, которые мне понравились в процессе разработки:

**Однофайловые компоненты.** Каждый компонент - отдельный `.vue`-файл,
разделённый на три секции: `<script setup>`, `<template>` и `<style scoped>`.
Такое разделение удобно: вся логика, разметка и стили одного компонента находятся рядом, но при этом не перемешиваются.
Атрибут `scoped` у стилей гарантирует, что CSS-правила не вытекут в другие компоненты,
что гарантирует модульность и предотвращает конфликты стилей.

**Реактивность: `ref` и `computed`.** Реактивная система Vue оказалась очень интуитивной. `ref` используется для
примитивных значений, а `computed` - для производных данных. В `ObserveMiners.vue` из одного
словаря шахтёров `miners` через `computed` автоматически вычисляются три отдельных списка: незапущенные,
работающие и истощённые. При каждом обновлении `miners` все три списка пересчитываются сами.

**Хуки жизненного цикла.** `onMounted` и `onUnmounted` позволили аккуратно управлять polling-интервалами.
В компонентах `Balance.vue` и `ObserveMiners.vue` при монтировании стартует `setInterval`,
который каждые 500 мс запрашивает свежие данные с бэкенда. При размонтировании компонента интервал очищается в
`onUnmounted` - утечки памяти исключены.

**`watch`.** В `Equipment.vue` используется `watch` для отслеживания вычисляемого свойства `allBought`:
как только все единицы оборудования куплены, автоматически открывается модальное окно с предложением перезапустить игру.

**Директивы шаблона.** `v-if` / `v-else`, `v-for`, `@click`, `:disabled`, `:class`- директивы мне показались
более удобными, чем подход в реакт.

### Vite

Vite - современный сборщик для веб-приложений, всегда его использую

---

### Axios

Axios - HTTP-клиент для браузера, использованный для обращений к REST API бэкенда.
Пользоваться им удобнее, чем стандартным fetch за счет удобного получения json из него

---

### Docker & Docker Compose

Docker позволяет упаковать бэкенд и фронтенд в контейнеры,
а с помощью compose можно поднять всё одной командой (`docker compose up`).

---

### Go

Go - язык, на котором написан бэкенд.

---

### gorilla/mux

`gorilla/mux` - HTTP-роутер для Go, использованный для построения REST API. Он позволил разделить маршруты
по методам (`GET`/`POST`) и организовать цепочки middleware-суброутеров: первый слой (`HasStartedMiddleware`) блокирует
запросы до запуска шахты, второй (`PauseMiddleware`) — во время паузы. Такой подход сделал логику защиты маршрутов
декларативной и переиспользуемой.

## Заключение

Проект получился небольшим, но позволил попробовать новую технологию - Vue 3.

Основной целью было знакомство с Vue 3, и оно оказалось приятным. Порог входа заметно ниже, чем у React:
реактивность работает предсказуемо, шаблоны читаются естественно, однофайловые компоненты держат код аккуратным.

Еще мне очень понравился легко читаемый синтаксис.
